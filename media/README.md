# Media Directory — SunBiz Funding Ad Creatives

This directory is the central store for all ad creative assets used in the SunBiz Funding
marketing system. Images are generated by AI, tracked in a registry, and consumed by the
publishing pipeline.

## Directory Structure

```
media/
├── assets/          — Brand assets (logo, fonts, color palettes, templates)
├── exports/         — AI-generated ad creatives, organized by campaign
│   └── {campaign_name}/
│       ├── ad_{concept}.png
│       └── ...
├── raw/             — Raw/unprocessed media inputs (photos, brand source files)
└── ASSET_REGISTRY.md — Single source of truth for all generated creatives
```

## Publishing Pipeline

### 1. Image Generation

`scripts/imagen_generate.py` calls the Gemini API and writes PNG files to
`media/exports/{campaign_name}/`. After each successful save the script calls
`register_asset()`, which appends a new row to `ASSET_REGISTRY.md` with status `DRAFT`.

```
imagen_generate.py
    └── generate_ad_image()  →  media/exports/{campaign}/ad_{concept}.png
                             →  ASSET_REGISTRY.md  (new DRAFT row appended)
```

### 2. Review and Approval

A human reviews generated creatives and changes their status in `ASSET_REGISTRY.md`
from `DRAFT` to `READY`. Only `READY` assets are picked up by the publishing steps below.

### 3. Publishing via Late MCP or Meta Marketing API

When running a publish job the system reads `ASSET_REGISTRY.md`, filters for rows where
`Status = READY`, and uploads those files to the target platform.

- **Late MCP server** — used for scheduled social media posts (LinkedIn, Instagram, etc.)
- **Meta Marketing API** — used for paid Meta Ads (Facebook / Instagram feed and stories)

After a successful upload the `Status` column is updated to `PUBLISHED` and the
`Published` column is filled with the platform and publish date, e.g.
`Meta Ads 2026-03-11`.

### 4. Creative Refresh Policy

Ad fatigue degrades performance after 2–3 weeks. Assets whose `Created` date is older
than 21 days and whose `Status` is `PUBLISHED` should be flagged for replacement.
Generate new variants, set them to `READY`, and `ARCHIVE` the old entries once the new
creatives are live.

## Status Reference

| Status | Meaning |
|--------|---------|
| DRAFT | Generated but not yet reviewed |
| READY | Approved — eligible for publishing |
| PUBLISHED | Live on at least one platform |
| ARCHIVED | Retired or replaced; no longer published |

## Naming Convention

Generated files follow the pattern `ad_{concept}_{size}.png` inside their campaign folder:

```
media/exports/consolidation_q2_2026/
    ad_before_after_1080x1080.png
    ad_roadmap_1080x1080.png
    ad_payment_table_1080x1350.png
```

Campaigns are named `{topic}_{quarter}_{year}` or a short descriptor such as
`test_consolidation_v1` for early test runs.
